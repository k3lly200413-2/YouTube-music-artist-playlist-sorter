# YouTube-music-artist-playlist-sorter
A small Python tool that takes your YouTube Music library playlists and creates
sorted copies of them, ordered alphabetically by artist name.

For each playlist in your library, the script creates a new playlist named
<original title>_ordinata containing the same tracks, sorted by artist.
Your original playlists are left untouched.

## Requirements
- Python 3.11 or 3.12 (some dependencies used don't support 3.14 )
- [youtubemusicapi](https://ytmusicapi.readthedocs.io/en/stable/)

Install dependencies:
```bash
pip install ytmusicapi
```
## Setup / Authentication

This script authenticates using `browser.json` file, which stores your YouTube Music session headers.
1. Generate it once with
   ```bash
    python -c "from ytmusicapi import YTMusic; YTMusic.setup(filepath='browser.json')"
   ```
**NOTE** This file needs to be in the same directory as the .py file, otherwise the program will throw an error signaling a wrong json file being used.
Once this command is run it will prompt you to paste the request headers copied from your browser's dev tools while logged into music.youtube.com (see the [ytmusicapi authentication guide (https://ytmusicapi.readthedocs.io/en/stable/setup/browser.html) for the exact steps).
3. **Do Not Commit `browser.json`**, this file contains your session credentials. The gitignore in this project already contains the necessary entry to ignore the .json extention but it's still good to note
> Note: browser-based auth headers expire periodically (typically after
> logging out or after some time). If you start getting 401/403 errors,
> regenerate `browser.json`.

## Usage

```bash
python app.py
```

This script will:
1. Fetch all playlists in your library.
2. Skip all YouTube autogenerated playlists as well as any playlist which has already been sorted;
3. create a new playlist named "[copied_playlist_name]_ordinata", this is because the user this was orignially made for was italian, so it was more clear to write it in italian;
4. fetch all tracks in a playlist ( using `limit=None` allows us to fetch all songs and not be limited to the first 100 as default )
5. Sort all these tracks in alphabetical order of the artists name
6. Remove any duplicate tracks, this cause issues while testing so I decided to remove the duplicates directly to avoid issues with the YouTube backend
7. Finally add songs in bactches of 50 to avoid sending too many requests to the backend

## Known issues / things to watch for
 
- **YTMusicServerError: HTTP 409 Conflict** — occasionally happens when
  adding items right after creating a playlist, or if duplicate video IDs
  are sent in the same batch. The script retries failed batches up to 3
  times with backoff; persistent failures on a specific playlist are logged
  and skipped rather than crashing the whole run.
- **Large playlists take a while** — playlists with 500+ tracks take longer both to fetch (multiple paginated
  requests) and to populate (batched writes with pauses between them to
  avoid rate limits).
- **Auto/radio playlists aren't real playlists** — anything with an
  auto-generated ID (`LM`, `SS`, `RD...`) doesn't have a stable, ownable
  track list you can copy this way, hence the skip logic.
## Possible future improvements
 
- Sort by multiple keys (e.g. artist, then album, then track number).
- Option to update an existing `_ordinata` playlist in place instead of only
  creating new ones.
- Config file for privacy status, chunk size, instead of hardcoding in the script.
- Progress bar instead of printed status lines for large playlists.

## Disclaimer
 
This uses an unofficial API (`ytmusicapi`) that talks to YouTube Music's
internal endpoints. It isn't an official Google/YouTube product, and
excessive or rapid API calls could trigger temporary rate limiting on your
account. Use reasonably.
